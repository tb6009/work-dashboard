# 063 workDashboard — 주간 업데이트 가이드

> **목적**: 매주 월요일에 지난주 작업 데이터를 대시보드에 반영하는 표준 절차
> **업데이트 빈도**: 주 1회 (월요일 오전)
> **소요 시간**: 약 15~20분 (자동 추출 1분 + Claude 보강 10분 + 검토 5분)

---

## 1. 전체 아키텍처

```
docs/logs/YYYYMMDD_log.md          (일별 작업 로그 — 손으로 작성)
PROJECTS.md                          (마스터 대시보드)
각 프로젝트 PROJECT.md               (프로젝트별 상태)
        │
        ▼
[ Tier 1: 자동 추출 ]                scripts/extract-week.mjs
   ↓ 기계적 데이터 (날짜·파일수·결정·신규)
_drafts/YYYY-WXX_draft.json
        │
        ▼
[ Tier 2: AI 보강 ]                  .claude/commands/주간업데이트.md
   ↓ 의미적 데이터 (요약·blockers·nextAction)
app/src/data/weekly/YYYY-WXX.json
app/src/data/projects/[id].json (활동 있던 프로젝트만)
        │
        ▼
[ Tier 3: 배포 ]                     git commit + Vercel auto-deploy
```

**Tier 분리 이유**: 기계적 데이터는 스크립트로(빠르고·정확), 의미적 데이터는 Claude로(맥락·뉘앙스). 둘을 섞으면 오류가 늘고 비용도 증가.

---

## 2. 주간 업데이트 절차 (월요일 루틴)

### Step 1 — 자동 추출 실행 (1분)

```bash
cd /Users/jinhyunpark/Documents/cloude_Code/063_workDashboard
node scripts/extract-week.mjs 2026-W19
```

**스크립트 동작**:
1. 인자 `2026-W19` → 월~일 날짜 범위 계산 (`2026-05-04 ~ 2026-05-10`)
2. `../docs/logs/2026050{4..10}_log.md` 읽기
3. 각 로그에서 멘션된 프로젝트 ID 추출 (정규식)
4. 일별 파일 변경 수 추정 (로그 내 파일 목록 파싱)
5. `## 주요 결정사항` 섹션 추출 → `decisions[]`
6. `../PROJECTS.md`와 직전 주 weekly JSON 비교 → 신규 프로젝트 검출
7. 출력: `_drafts/2026-W19_draft.json`

**출력 예시 (mechanical only)**:
```json
{
  "week": "2026-W19",
  "range": { "from": "2026-05-04", "to": "2026-05-10" },
  "status": "current",
  "summary": "TODO",
  "kpis": { "activeProjects": 0, "sessions": 0, "newProjects": 0, "filesChanged": 0 },
  "daily": [...],
  "projects": [
    { "id": "061", "pct": 0, "filesChanged": 12, "imp": 0, "nextAction": "TODO" }
  ],
  "decisions": [...],
  "newProjects": [],
  "labelChanges": [],
  "buildAt": "..."
}
```

`TODO`·`0` 으로 표시된 필드는 Claude가 채움.

---

### Step 2 — Claude 보강 (10분)

VS Code에서 `063_workDashboard/` 열고 Claude 세션 시작:

```
/주간업데이트 2026-W19
```

이 슬래시 커맨드는 `.claude/commands/주간업데이트.md` 를 호출.

**Claude가 수행하는 일**:
1. `_drafts/2026-W19_draft.json` 읽기
2. 해당 주 raw 로그 다시 읽어 의미적 필드 채우기:
   - `summary` — 한 줄 핵심 요약
   - `kpis.sessions`, `kpis.activeProjects` 등 카운트
   - `projects[].pct` — 각 프로젝트 비중 (수동 평가 + AI 추정)
   - `projects[].imp` — 1~5 importance dots
   - `projects[].nextAction` — PROJECT.md 다음 액션에서 추출
3. `app/src/data/weekly/2026-W19.json` 으로 저장
4. **활동 있던 프로젝트**의 `app/src/data/projects/[id].json` 업데이트:
   - `currentSituation.summary` 갱신
   - `currentSituation.recentLogs` 에 새 로그 1건 prepend (최대 3건 유지)
   - 새 산출물 `deliverables[]` 에 추가
   - 완료된 `nextTasks[]` 제거, 새 next task 추가
5. `app/src/lib/data.ts` 의 `CURRENT_WEEK_ID` → `'2026-W19'` 로 갱신

---

### Step 3 — 검토 + 배포 (5분)

```bash
# 1. 로컬 확인
cd app && npm run dev
# http://localhost:3001 에서 시각적 확인

# 2. 차이 확인
git status
git diff app/src/data/weekly/2026-W19.json

# 3. 커밋 + 푸시
git add app/src/data/ app/src/lib/data.ts _drafts/
git commit -m "weekly: 2026-W19 update"
git push
# → Vercel 자동 배포 (~1분)
```

## 4. 신규 프로젝트 추가 시 (격주~월 1회)

신규 프로젝트가 시작되면 Step 2 전에 추가 작업:

1. `app/src/data/projects.json` 에 메타 한 줄 추가 (id·name·type·define·status·createdAt)
2. `app/src/data/projects/[id].json` 신규 생성
   - 템플릿: 기존 JSON 1개 복사 후 내용 교체
   - 필수 필드만 우선 채움 (recentLogs 1건, deliverables 2~3건, nextTasks 2~3건)
3. Step 2 의 `/주간업데이트` 가 알아서 weekly JSON에 반영

---

## 5. 영구 누적 원칙 (운영가이드 §9.6 따름)

- `app/src/data/weekly/*.json` — **영구 보존**, 삭제·덮어쓰기 금지
- `app/src/data/projects/*.json` — 갱신은 OK, 삭제는 금지
- `_drafts/*.json` — 임시 파일, 매주 새로 생성 (이전 draft 보존 권장)
- 3년 누적 시: 156개 weekly + 36개 monthly + 3개 yearly = 장기 성장 궤적 시각화 가능

## 6. 단계별 자동화 로드맵

| Phase | 자동화 수준 | 트리거 |
|---|---|---|
| **현재 (Phase 1)** | Tier 1 자동 + Tier 2 수동 호출 | 매주 월요일 사용자가 직접 |
| **Phase 2** | Claude Code Hook으로 월요일 자동 알림 | UserPromptSubmit hook |
| **Phase 3** | Claude Cron 매주 월요일 09:00 | `schedule` 스킬로 routine 등록 |
| **Phase 4** | Monthly·Yearly 집계 자동화 | 매월 1일 / 매년 1월 1일 |

## 7. 트러블슈팅

### "스크립트가 0건 추출함"
- 해당 주 로그 파일이 없거나 비었음
- 확인: `ls ../docs/logs/2026050*.md`
- 해결: 로그 작성 먼저 → 다시 실행

### "Vercel 배포 실패"
- 보통 JSON 문법 오류 (trailing comma 등)
- `npm run build` 로 로컬 검증
- VSCode JSON validation 켜기

### "프로젝트 ID 멘션이 잘못 추출됨"
- 정규식이 `061`을 `61`로 잘못 매칭하는 케이스
- `extract-week.mjs` 의 `PROJECT_ID_RE` 정렬 (긴 ID 먼저 매칭)
